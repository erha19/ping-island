## 亮点
- 新增 Claude Desktop 本地会话监控，能识别 Claude Desktop / local-agent 回合完成并同步到 Island 会话列表。
- 低功耗策略更完整，空闲、锁屏、安静时段会降低后台轮询、事件监听和 UI 动画开销，活跃会话仍保持响应。
- 设置页首屏更轻，重型分区会延后加载并显示加载态；集成配置目录入口现在直接打开对应目录，排查 hook 配置更顺手。

## 修复
- 修复 Cursor hook 会话把工作区识别成 `~/.cursor` 的问题；现在会从 Cursor transcript 路径恢复真实项目目录，列表不再显示 `.cursor`，点击也优先回到已打开的对应 Cursor 工作区窗口。
- 修复 Cursor hook 安装中的路径和事件名兼容问题，保持 Cursor / Claude 兼容 hook 事件语义一致。
- 修复 Codex 后台辅助会话过滤和跳转路径问题，减少空白页面和标题生成线程误入主列表的情况。
- 远程 SSH bridge 会在远端二进制已存在时跳过重复 SCP，并在需要时通过 SSH 管道回退，提高远程连接可靠性。

## 说明
- Sparkle build 号提升到 `41`，版本号提升到 `0.13.1`。
- 这是基于 `0.13.0` 的稳定性与体验修复版本，重点覆盖 Cursor 会话追踪、Codex 会话过滤、Claude Desktop 监控、远程连接和低功耗策略。

## Highlights
- Added Claude Desktop local-session monitoring so Claude Desktop / local-agent turn completion can appear in the Island session list.
- Expanded the low-power policy so quiet, locked, and idle periods reduce background polling, event monitoring, and UI animation work while active sessions stay responsive.
- Made Settings faster on first paint by deferring heavier sections behind loading states; integration configuration directory actions now open the target folder directly.

## Fixes
- Fixed Cursor hook sessions being associated with `~/.cursor`. Ping Island now recovers the real project workspace from Cursor transcript paths, stops showing `.cursor` in the session list, and prefers focusing the already-open Cursor workspace window.
- Fixed Cursor hook install path and event-name compatibility so Cursor and Claude-compatible hook semantics stay aligned.
- Fixed Codex auxiliary-session filtering and jump navigation issues, reducing blank pages and background title-generation threads leaking into the main list.
- Remote SSH bridge setup now skips duplicate SCP when the remote binary already exists and falls back through an SSH pipe when needed.

## Notes
- The Sparkle build number is now `41`, and the app version is now `0.13.1`.
- This is a stability and experience patch on top of `0.13.0`, focused on Cursor session tracking, Codex session filtering, Claude Desktop monitoring, remote reliability, and low-power behavior.
