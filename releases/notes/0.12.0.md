## 亮点
- 全局快捷键现在可以真正清空并保持禁用，重启后不会再恢复默认的 Option-J / Option-L。

- 修正快捷键重新注册时的 Carbon hotkey ID 管理，避免旧注册误路由到当前动作表。
- 调整 app icon 相关显示尺寸，让菜单栏和发布表面中的图标观感更克制。

## Highlights
- Global shortcuts can now stay cleared and disabled; relaunching the app no longer restores the Option-J / Option-L defaults after a user clears them.
- AskUser input is steadier: hover panels stay open while text is focused or drafted, unsent text is not lost when the pointer leaves, and Return submits through the same path as the send button.

## Fixes
- Claude Code CLI `Stop` events now return sessions to idle instead of marking them as waiting for input, reducing false attention states after a turn finishes.
- Claude Code sessions now have PID / orphan detection plus same-directory stale session cleanup, preventing crashed, closed, or restarted sessions from staying stuck in processing.
- The session display layer deduplicates by provider + cwd to reduce duplicate rows created by resume or concurrent hook events.
- Shortcut re-registration now uses fresh Carbon hotkey IDs so old registrations cannot route into the current action table.
- App icon sizing was refined so the icon feels less oversized in menu bar and release surfaces.

## Notes
- The Sparkle build number is now `39`, and the app version is now `0.12.0`.
- This release focuses on shortcut persistence, AskUser input stability, and Claude Code session state reliability on top of `0.11.2`.
