## 亮点
- 新增 Kimi CLI 官方 hooks 支持，Ping Island 现在会管理 `~/.kimi/config.toml` 中的 `[[hooks]]` 条目，并保留 providers、models、loop_control 等非 Island 配置。
- README 和应用中的 Kimi 形象已统一为原先实现的蓝色键盘球，并加入 Kimi GIF 展示。
- 独立悬浮宠物新增尺寸模式，可按显示器自动放大，也可以保持紧凑或手动选择更大的悬浮形象。

## 修复
- 设置窗口隐藏后不再继续驱动宠物预览动画，降低再次打开设置时的卡顿和桌面宠物右键菜单卡死风险。
- Kimi `Stop` 会按单轮回复完成处理，只有 `SessionEnd` 才结束会话，避免把回合完成误当作会话关闭。
- App Store 版本移除了不适合审核路径的网络服务 entitlement 和 Accessibility 聚焦入口，保持 Developer ID 版本现有能力不变。

## 说明
- Sparkle build 号提升到 `40`，版本号提升到 `0.13.0`。
- 这是基于 `0.12.0` 的客户端覆盖、悬浮宠物体验、设置面板性能和 App Store 提交流程版本。

## Highlights
- Added official Kimi CLI hook support. Ping Island now manages `[[hooks]]` entries in `~/.kimi/config.toml` while preserving unrelated providers, models, loop_control, and other TOML configuration.
- Kimi documentation and app visuals now use the original blue keyboard-orb mascot, including the README GIF strip.
- Floating pet mode now has explicit size modes, including display-aware automatic scaling, the original compact size, and a larger option.

## Fixes
- Hidden settings windows no longer keep mascot preview animations running, reducing the reopen lag and desktop-pet right-click freeze risk.
- Kimi `Stop` is treated as an assistant-turn completion, while only `SessionEnd` closes the session.
- The App Store build removes review-sensitive network server entitlement and Accessibility focus entry points while keeping the Developer ID lane unchanged.

## Notes
- The Sparkle build number is now `40`, and the app version is now `0.13.0`.
- This release focuses on client coverage, floating pet polish, settings-panel performance, and App Store submission readiness on top of `0.12.0`.
