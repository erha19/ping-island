## 亮点
- 优化 Island 8-bit 默认音效：介入、完成、资源限制等阶段的提示音更贴近实际语义，已自定义的用户配置不会被覆盖。
- Claude Code 的 `Stop`、`SubagentStop`、`StopFailure` 等 hook 状态识别更准确，任务完成提示音、子 agent 运行和父会话存活状态更稳定。
- Codex 对安静刷新和用户中断的识别更稳：正常空闲刷新不再让运行中任务闪烁，中断后的工具状态会正确回到可继续处理的状态。

## 修复
- 修复 Always-Allow 自动批准场景下仍可能持续播放介入提示音的问题。
- 增加会话存活扫描，清理进程已经退出但结束 hook 没有送达的僵尸会话。
- 锁定 Xcode SwiftPM 依赖并放宽少量 CI 时间敏感测试，降低发布验证中的偶发失败。

- Sparkle build 号提升到 `48`，版本号提升到 `0.16.0`。

## Highlights
- Refined Island 8-bit default sounds so attention, completion, and resource-limit events better match their meaning without overriding existing user customizations.
- Improved Claude Code hook-state recognition for `Stop`, `SubagentStop`, and `StopFailure`, making completion sounds, subagent turns, and parent-session liveness more reliable.
- Stabilized Codex quiet refreshes versus user-aborted turns: normal idle refreshes no longer make active work flicker, while interrupted tools return to an actionable idle state.

## Fixes
- Fixed repeated attention sounds when Always-Allow auto-approval is enabled.
- Added a session liveness sweep to remove stale sessions when a process exits without delivering its end hook.
- Locked Xcode SwiftPM dependencies and relaxed a small CI timing edge to reduce release-validation flakes.

## Notes
- The Sparkle build number is now `48`, and the app version is now `0.16.0`.
- This release focuses on sound behavior plus Codex and Claude Code task-progress recognition.
